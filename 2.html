Fonctionnement d'Internet

a. Quelles infrastructures permettent l’existence d’internet et la transmission des données ?

Les infrastructures qui permettent l'existence d'internet et la transmission des données sont la couche physique :
c'est la couche des infrastructures des fibres optiques transocéaniques, IXP (infrastructure essentielle où les réseaux
se réunissent pour se connecter et échanger du trafic sur internet). La couche commande et contrôle: c’est la couche
où l'on retrouve le DNS (système de Nom de Domaine) le BGP (protocole permettant l'échange d'information de routage
dans un réseaux de système autonomes) les tables de protocoles de routage et les logiciels qui les implémentes. La couche
logique : c'est la couche des données publiées, des logiciels qui permettent d'y accéder et de les transformer ou encore
la couche des serveur web, des moteurs de recherche et des navigateurs.

b. Comment s’effectue la communication d’un ordinateur à un autre sur internet ?

La communication d’un ordinateur à un autre sur internet se sert de la technologie TCP / IP, qui est une suite de protocole
utilisés pour le transfert de données sur internet. Mais avant d’expliquer comment se fait la communiquer il faut d’abord
définir quelque élément. Chaque application est définie par un port de destination, par exemple le WEB est défini par le
port 80, ce port permet aux ordinateurs de savoir à quelle application est adressé. Chaque machine est identifiée par une
adresse IP, ainsi une adresse IP est unique et elle permet de différencier les machines entre elles et donc pouvoir envoyer
le paquet au près du bon destinataire. Le paquet est un bloc de données formaté envoyé sur un réseau. Le routeur permet
d’interconnecter des réseaux entre eux. Ainsi à présent nous pouvons présenter la communication sur internet. Elle se fait
sur 4 couches, couche 4 : application, couche 3 : transport, couche 2 : internet, couche 1 : réseau. Chaque couche a une
tâche bien définie permettant le bon transport des données.

  1. Couche 4 : Cette couche traduit en langage machine les messages qui seront envoyés sur internet.

  2. Couche 3 : Elle encapsule les données, c’est-à dire l’ajout d’un entête, le port de destination et le port source (le numéro qui identifie les applications) et donc ajouté. Le message devient ainsi un segment.

  3. Couche 2 : Elle encapsule de nouveaux les données mais cette fois ci rajoute l’adresse IP de destination et l’adresse IP source (adresse IP de notre ordinateur). C’est lors de cette étape que le segment devient un paquet.

  4. Couche 1 : Cette couche achemine les paquets de routeur en routeur jusqu’au destinataire. C’est ainsi que se fait la communication d’un ordinateur à un autre, mais le protocole est le même d’un ordinateur à un serveur.
